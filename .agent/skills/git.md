# Git Workflow

## Commit Messages
- Use the Conventional Commits format if possible (but we are lenient for this project).
- Format: `<type>: <description>`
  - `feat`: New feature (e.g., new page, new component).
  - `fix`: Bug fix (e.g., fixing a layout issue).
  - `docs`: Documentation changes.
  - `style`: Formatting, missing semi-colons, etc.
  - `refactor`: Code change that neither fixes a bug nor adds a feature.
  - `chore`: Configuration changes (like this setup).

## Branching
- Direct commits to `main` (or `master`) are acceptable for this solo/small project setup unless otherwise specified.
